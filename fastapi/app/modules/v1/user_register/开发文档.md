## 📖 接口文档

### 用户注册模块 API 文档

**基础信息：**
- 服务地址：http://localhost:8100
- API版本：v1
- 模块前缀：/api/v1/user_register

#### 1. 用户注册接口

**接口地址：** `POST /api/v1/user_register/register`

**请求参数：**
```json
{
  "username": "string",    // 用户名，3-20字符，只能包含字母数字
  "email": "string",       // 邮箱地址，必须是有效邮箱格式
  "password": "string"     // 密码，8-50字符，必须包含字母和数字
}
```

**成功响应（201）：**
```json
{
  "id": 1,
  "username": "alice",
  "email": "alice@example.com",
  "is_active": true,
  "created_at": "2025-09-29T03:25:10"
}
```

**错误响应：**
- `400` - 用户名已存在：`{"detail": "用户名已存在"}`
- `400` - 邮箱已存在：`{"detail": "邮箱已存在"}`
- `422` - 数据验证错误：详细验证信息
- `500` - 服务器内部错误

#### 2. 模块测试接口

**接口地址：** `GET /api/v1/user_register/test`

**响应：**
```json
{
  "message": "用户注册模块正常运行",
  "module": "user_register",
  "version": "v1"
}
```

### 技术架构

**核心技术栈：**
- FastAPI 0.104.1
- SQLAlchemy 2.0.23
- MySQL + PyMySQL
- Pydantic（数据验证）
- SHA256+盐值（密码加密）

**架构特点：**
- ✅ 模块化设计：每个功能独立模块
- ✅ 版本化管理：支持多版本并存
- ✅ 自动注册：新模块零配置自动发现
- ✅ 零修改原则：新功能不修改现有代码

### 数据库设计

**表名：** `us_users`（us_前缀避免冲突）

**表结构：**
```sql
CREATE TABLE us_users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_username (username),
    INDEX idx_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```
