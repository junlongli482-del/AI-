## 1. 项目概述

### 1.1 项目简介

FastAPI模块化文档管理系统是一个基于FastAPI框架的企业级文档管理平台，采用模块化架构设计，支持版本化开发，具备完整的用户系统、文档管理、AI集成、分享系统等功能。

### 1.2 技术架构特色

- **🏗️ 模块化架构**：每个功能独立成模块，支持热插拔
- **📦 版本化管理**：v1用户系统，v2文档系统，支持并行开发
- **🔄 自动注册**：新模块零配置自动发现和注册
- **🛡️ 零侵入原则**：新增功能不影响现有功能
- **🤖 AI服务集成**：统一AI客户端，支持多种AI服务
- **🧪 标准化测试**：API响应格式预测试 + 完整功能测试

### 3.4 数据库设计

#### 3.4.1 用户系统表（v1）

```sql
CREATE TABLE us_users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    username VARCHAR(50) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    nickname VARCHAR(50) DEFAULT NULL UNIQUE,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

#### 3.4.2 文档系统表（v2）

```sql
-- 文件夹表
CREATE TABLE us_folders (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    parent_id INT DEFAULT NULL,
    user_id INT NOT NULL,
    level TINYINT DEFAULT 1,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (parent_id) REFERENCES us_folders(id) ON DELETE CASCADE,
    FOREIGN KEY (user_id) REFERENCES us_users(id) ON DELETE CASCADE
);

-- 文档表
CREATE TABLE us_documents (
    id INT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(200) NOT NULL,
    content LONGTEXT,
    file_path VARCHAR(500),
    file_type ENUM('md', 'pdf') NOT NULL DEFAULT 'md',
    file_size INT DEFAULT 0,
    summary TEXT,
    status ENUM('draft', 'published', 'review_failed') DEFAULT 'draft',
    publish_time TIMESTAMP NULL,
    review_message TEXT,
    folder_id INT DEFAULT NULL,
    user_id INT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,

#### 4.1.5 可选认证依赖实现 🔐 🆕

**问题描述**：
分享系统需要支持匿名和登录用户的混合访问模式，但FastAPI的`Depends(get_current_user)`即使声明为`Optional`仍然会强制执行认证检查。

**解决方案**：
```python
def get_optional_current_user(request: Request, db: Session = Depends(get_db)):
    """
    可选的用户认证依赖
    - 如果有有效token，返回用户对象
    - 如果没有token或token无效，返回None
    - 不会抛出认证异常
    """
    try:
        authorization = request.headers.get("Authorization")
        if not authorization or not authorization.startswith("Bearer "):
            return None
        
        # 使用现有的认证服务验证token
        token = authorization.split(" ")[1]
        user_data = auth_service.verify_token(token, db)
        
        if user_data:
            user = db.query(User).filter(User.id == user_data["user_id"]).first()
            return user if user and user.is_active else None
        
        return None
    except Exception:
        return None
```

**经验教训**：

**解决方案**：
```python
# 在services.py中
import os

def _build_share_response(self, share: DocumentShare, db: Session) -> ShareResponse:
    # 从环境变量读取基础URL，支持内网穿透
    base_url = os.getenv("BASE_URL", "http://localhost:8100")
    share_url = f"{base_url}/api/v2/share_system/public/{share.share_code}"
    
    return ShareResponse(
        # ... 其他字段
        share_url=share_url,
        # ...
    )
```
